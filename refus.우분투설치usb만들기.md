IT 전문가의 관점에서, **Rufus**는 윈도우 환경에서 리눅스 부팅 USB를 제작하는 데 가장 빠르고 신뢰할 수 있는 도구 중 하나입니다.

우분투(Ubuntu) 설치 USB를 만드는 상세 절차를 단계별로 안내하겠습니다.

---

### **Rufus를 이용한 우분투 설치 USB 제작 가이드**

#### **1단계: 준비물 확인 및 다운로드**

| 구분 | 내용 | 비고 |
| :--- | :--- | :--- |
| **USB 드라이브** | 8GB 이상의 USB 메모리 | **주의:** USB 내의 모든 데이터는 삭제됩니다. |
| **Rufus** | 최신 버전의 **[Rufus](https://rufus.ie/ko/)** 실행 파일 (설치 불필요) | 간편하게 포터블 버전을 다운로드하세요. |
| **우분투 ISO 파일** | **[우분투 공식 웹사이트](https://ubuntu.com/download/desktop)**에서 최신 LTS(장기 지원) 버전의 `.iso` 이미지 파일 다운로드 | 안정성이 높으므로 LTS 버전 권장 |

#### **2단계: Rufus 실행 및 장치 설정**

1.  준비된 USB 드라이브를 컴퓨터에 연결합니다.
2.  다운로드한 **Rufus 실행 파일**을 실행합니다.

| 설정 항목 | 선택 내용 | IT 전문가 팁 |
| :--- | :--- | :--- |
| **장치 (Device)** | 연결한 USB 드라이브를 정확히 선택 | 용량과 드라이브 문자를 통해 실수하지 않도록 확인 |
| **부트 선택 (Boot selection)** | **[선택]** 버튼을 눌러 다운로드한 **우분투 ISO 파일**을 지정 | |

#### **3단계: 파티션 스키마 및 파일 시스템 설정**

이 부분이 가장 중요하며, 설치할 컴퓨터의 환경에 맞춰 정확히 설정해야 부팅 문제가 발생하지 않습니다.

| 설정 항목 | 권장 설정 | 상세 설명 및 강조 |
| :--- | :--- | :--- |
| **파티션 방식 (Partition scheme)** | **GPT** | **(강조)** 2010년 이후에 출시된 대부분의 최신 PC는 **UEFI** 펌웨어를 사용합니다. UEFI 환경에서는 **GPT** 방식이 표준이며 보안성도 높습니다. 특별히 구형 PC에 설치하는 경우가 아니라면 GPT를 선택하세요. |
| **대상 시스템 (Target system)** | **UEFI (CSM 제외)** | GPT를 선택하면 자동으로 UEFI가 선택됩니다. |
| **볼륨 레이블 (Volume label)** | 자유롭게 입력 (예: `Ubuntu_LTS`) | 설치 후 부팅 메뉴에서 알아보기 쉽도록 설정 |
| **파일 시스템 (File system)** | **FAT32** | 대부분의 시스템에서 가장 호환성이 높은 표준 파일 시스템입니다. |

#### **4단계: 부팅 디스크 생성 시작**

1.  모든 설정이 완료되었으면 하단의 **[시작]** 버튼을 클릭합니다.
2.  잠시 후, **"ISOHybrid 이미지 발견"** 팝업이 나타나면 **'ISO 이미지 모드로 쓰기 (권장)'**를 선택하고 **[OK]**를 클릭합니다. (대부분의 경우 이 옵션이 안정적입니다.)
3.  **경고** 팝업(USB 데이터 삭제)이 뜨면 내용을 확인 후 **[확인]**을 눌러 진행합니다.

#### **5단계: 완료**

* 진행 상태 바가 모두 채워지고 상태 메시지가 **'준비 (READY)'**로 바뀌면 부팅 USB 제작이 완료된 것입니다.
* **[닫기]**를 클릭하고 USB를 안전하게 제거합니다.

---

### 🚀 실전 팁: 영구 저장 공간 (Persistent Partition) 활용

강의나 테스트 목적으로 우분투 USB를 사용할 경우, Rufus 설정에서 **'영구 파티션 크기(Persistent partition size)'** 옵션을 0이 아닌 값으로 설정할 수 있습니다.

* 이 옵션을 설정하면, USB로 부팅하여 사용하면서 생성한 파일, 설치한 프로그램, 변경된 시스템 설정 등이 **재부팅 후에도 그대로 보존**됩니다.
* 이 기능은 단순한 설치 디스크를 넘어, **휴대 가능한 개발 환경**을 구축할 때 매우 유용합니다. (다만, 우분투의 경우 특정 버전에서만 지원되거나 과정이 복잡할 수 있어, Rufus가 지원할 경우에만 활용하는 것을 권장합니다.)
